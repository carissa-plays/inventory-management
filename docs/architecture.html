3Data Flow — the Filter System

1
User adjusts a filter
Time Period, Warehouse, Category, or Order Status — bound to module-level refs in useFilters.js, shared (singleton) across every component that imports it.
2
View requests data
Calls api.js with getCurrentFilters(); filters become query params (warehouse, category, status, month) — a value of 'all' means "no filter".
3
FastAPI route receives the request
Endpoint in main.py re-applies the same filter logic server-side via apply_filters() and filter_by_month() (quarters like Q1-2025 or direct months like 2025-01).
4
Result validated against a Pydantic model
Response shape is enforced by response_model before serialization — keeps the JSON contract consistent with the frontend's expectations.
5
Vue renders derived state
Raw response lands in a ref (e.g. allOrders); views compute filtered/aggregated values from it via computed() rather than storing duplicate filtered lists.

4Notable Characteristics

No time dimension on inventory

/api/inventory ignores the month filter — only orders, backlog-adjacent, and dashboard data are date-aware.

Auth is fully mocked

useAuth.js hardcodes an authenticated user; there is no login flow or server-side auth check anywhere in the stack.

Stateless backend

All data loads into memory at startup from JSON files; nothing written during a request persists past a server restart.

Public repository

No secrets, internal hostnames, or private registry URLs should ever be committed — this repo and any fork are public.
